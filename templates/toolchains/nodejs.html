Predictable NodeJS
release cadence on Ubuntu

Ubuntu's predictable release cadence makes it the ideal platform for building and running Node.js applications. Each Ubuntu LTS release includes a Canonical-maintained Node.js LTS version, giving you access to the latest features while ensuring long-term security and stability.

Here's how it works:

  • Starting with Ubuntu 14.04, every Ubuntu LTS release has included a Canonical-maintained Node.js LTS version.
  • Each Node.js version receives security updates through Ubuntu Pro, offering at least 10 years of coverage. This extends beyond the upstream Node.js maintenance window from the OpenJS Foundation.
  • Developers benefit from consistent, production-grade environments that match their local setups. This removes surprises at deployment time.

Security through confinement, when installing Node.js as a snap package

Using the Node.js snap gives you:

  • The ability to install multiple Node.js versions in parallel, without conflicts
  • Isolated, sandboxed execution for better security
  • Automatic background updates with rollback support
  • Granular control over updating the snap package through Landscape

The Node.js snap is packaged and published by the OpenJS Foundation. It is trusted, up-to-date, and secured by OpenJS - but does not get 12 years of security coverage through Ubuntu Pro.

FIPS compliant cryptography in certified Node.js builds

FIPS compliance

Organizations in need of a Node.js binary with FIPS-compliant cryptography will never need to compile Node.js with these capabilities from source. Canonical publishes a FIPS-compliant Node.js binary for customers who enable FIPS on Ubuntu. The option to enable FIPS in 1 command is included with Ubuntu Pro.

If FIPS has been enabled in Ubuntu through the Ubuntu Pro Client, the FIPS-compliant version of the Node.js package will be installed automatically.
