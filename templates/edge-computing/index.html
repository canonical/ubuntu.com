What are edge computing use cases?

Real-life applications are localized — in a factory, a shop, a theatre, a stadium, on the subway, even in motion. Edge computing has the potential to transform almost any industry, serving workloads where and when needed. What are micro clouds?

{%- endif -%} {%- if slot == 'list_item_description_1' -%}
{{ image(url="https://assets.ubuntu.com/v1/d694b804-edge-computing-updated.png", alt="Edge computing micro cloud architecture", width="1800", height="1201", hi_def=True, loading="lazy", attrs={"class": "p-image-container__image"}) | safe }}

Edge computing isn’t a solution: it’s a term for describing geographically-distributed computing. Micro clouds are a solution for edge computing.

A micro cloud is a new class of infrastructure for on-demand computing at the edge. Micro clouds are small footprint clusters of compute nodes with local storage and secure networking, optimized for repeatable, reliable remote deployments.

Standard and open APIs provide the same functionality you would expect from larger clouds, and an abstraction layer separates the hardware from the software.

{%- endif -%} {%- if slot == 'list_item_title_2' -%}

Advantages of micro clouds for edge computing

{%- endif -%} {%- if slot == 'list_item_description_2' -%}

Micro cloud enables you to compose customized edge computing solutions from the best open source components, anywhere and at scale.

  • Flexibility and scalability
  • Full-stack automation
  • Open and standard APIs
  • Abstraction layer

The edge of the network is the center of everyday businesses and applications. Micro clouds empower ecosystems rather than single-vendor solutions, bringing apps closer to the consumers.

{%- endif -%} {%- if slot == 'list_item_title_3' -%}

What is the size of a micro cloud?

{%- endif -%} {%- if slot == 'list_item_description_3' -%}

Micro clouds go from tiny Kubernetes clusters in an outdoor box to small private edge clouds localized nearest to the consumers of an application. They can extend in site size, scale number of sites, and run with no on-site maintenance.
