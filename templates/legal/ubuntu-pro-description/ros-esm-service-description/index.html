Specialist Services for ROS ESM

As a Robot Operating System (ROS) Extended Security Maintenance (ESM) customer, you are entitled to the benefits and services, subject to the exclusions and conditions, as defined in the Ubuntu Pro Service Description.

ROS scope is included in Ubuntu Pro. As an Essential, Standard or Advanced customer, you are also entitled to the benefits and services for ROS as described below.

ROS ESM Service and ROS Support Scope

  1. ROS ESM Service

    1. ROS ESM Service provides:

      1. Available High and Critical CVE fixes for a number of core ROS packages for ROS 1 Kinetic and Melodic, and ROS 2 Foxy. This includes packages in the REP-142 'ros_base' and a majority of the 'desktop' metapackages for each ROS distribution.

    2. ROS ESM Service expressly excludes:

      1. Security fixes for CVEs that are not High or Critical.
      2. Managed Services, as per defined in the Ubuntu Pro Service Description, for ROS or robotic applications.

  2. ROS Support Scope

    As a Standard or Advanced customer, you are also entitled to the benefits and services for ROS ESM and the support described below.

    1. Support requests will follow the Support Services Process as defined in the Ubuntu Pro Service Description.
    2. Support requests will follow the Support Severity Levels definition as defined in the Ubuntu Pro Service Description.
    3. Support requests for third party and abandoned packages will be categorized as Severity Level 4.

    4. ROS Support Exclusions

      1. Canonical does not guarantee a work-around, resolution or resolution time.
      2. Bug fixes for packages in the end of life release as defined in ROS release cycles, unless a bug was created by an Extended Security Maintenance security update.
      3. Support for third party packages that have been abandoned.
      4. Support for the installation and configuration of ROS upstream.
      5. Development support for ROS.
      6. Maintenance, configuration and management of robots.
      7. Configuration files, including YAML/XML-launch.
      8. Support for parts of packages containing URDF/SRDF.
